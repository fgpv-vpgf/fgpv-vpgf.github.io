(inner) getLayerInsertPosition(sourceId, targetId) → {Number}

Source:
  • geo/layer-registry.service.js, line 263

Finds a position at which to insert the source layer so it's positioned directly above target layer (if one specified). If the target layer is no specified, the source layer is placed at the bottom of its sort group.

NOTE the ESRI map stack does not reflect the legend and is arranged in reverse order for ESRI low index = low drawing order; legend: low index = high drawing order. See design notes in https://github.com/fgpv-vpgf/fgpv-vpgf/issues/514 for more details.

Parameters:
Name Type Description
sourceId String

the id of the layer to be moved
targetId String

the id of the layer the target layer will be moved on top of; can be -1, if its the end of the list
Returns:

index at which the source layer should be inserted in the map stack

Type
Number

(inner) moveLayer(sourceId, targetId)

Source:
  • geo/layer-registry.service.js, line 308

Move a source layer within the map on top (visually) of the target layer.

NOTE this does not modify the legend, movement within the legend should be handled separately, ideally calling this function immediately before or after the legend is updated.

IMPORTANT NOTE: targetId must be the id of the layer which is actually in the map stack; this can't be a placholder which is not added to the map object.

Parameters:

sourceId String

the id of the layer to be moved
